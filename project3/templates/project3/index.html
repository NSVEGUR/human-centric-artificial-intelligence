Task 3

Learning-to-Defer

The system learns when to predict itself and when to defer to the best available expert. The deferral rule follows the Bayes-optimal decision: defer if the classifier's uncertainty exceeds the expected expert error, i.e., 1 − max P(y|x) > 1 − P(expert correct | x). Sweeping a scale parameter α generates the full accuracy vs. coverage curve.

How it works

For each test instance, the system computes P(expert correct | x) = Σk P(y=k|x) × accuracyexpert,k. The best expert is selected per-class. Deferral happens when the classifier's uncertainty (1 − max P(y|x)) exceeds the expert's expected error (1 − P(expert correct | x)). The parameter α scales this threshold: α=0 always defers, α→∞ never defers, α=1 is Bayes-optimal.

Task 4

Active Learning for Expert Competence Discovery

We now assume no expert labels are available initially. An active learner queries the expert on selected instances to learn the expert's competence profile (per-class accuracy), then uses that to build a deferral policy.

Setup: {{ al_stats.n_pool }} unlabeled pool instances, {{ al_stats.n_eval }} held-out evaluation instances. Four query strategies from the lecture are compared. Expert competence is estimated with Laplace smoothing (prior: 50% accuracy, 10 pseudo-counts).

Random

Naive baseline. Queries uniformly at random — provides balanced class coverage.

Least Confidence

uLC(x) = 1 − maxk P(k|x). Queries where classifier is least confident.

Margin Sampling

uM(x) = 1 − (P(ŷ₁|x) − P(ŷ₂|x)). Queries where top-2 classes are ambiguous.

Entropy

uEnt(x) = −Σk P(k|x) log P(k|x). Full distribution uncertainty measure.

Key insight

Uncertainty-based strategies (Least Confidence, Margin, Entropy) focus on hard examples near the decision boundary, which may be concentrated in certain classes. This biased sampling can slow expert competence discovery compared to random sampling, which covers all classes more evenly.
