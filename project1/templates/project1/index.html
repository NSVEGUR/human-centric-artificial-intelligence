Welcome — here is what this tool does and how to use it

This page walks you through a complete machine learning workflow from start to finish. You start by uploading a CSV file, the tool figures out whether it is a classification or regression problem, and then you can explore your data visually, train several different models, and compare how well they perform. Everything you see is interactive — there are no static images anywhere on this page.

Once your data is loaded you will see three charts. The scatter plot lets you pick any two features and see how your data points spread out, coloured by their class or value. The histogram shows you the distribution of a single feature so you can spot skew, outliers, or overlap between classes. The correlation matrix is the best place to start — it shows at a glance which features are strongly related to each other and which are independent, which helps you pick the most informative pair to look at in the scatter plot.

You can interact with every chart by hovering over any point or cell to see its exact values. Drag a rectangle on a chart to zoom into a busy area, and double-click anywhere to zoom back out. {% if problem_type == 'classification' %} Each class appears separately in the chart legend — click a class name once to hide it, and double-click it to show only that class. This is really useful when classes overlap and you want to focus on just one or two of them. {% endif %} The dropdowns above each chart let you switch which features you are looking at and the chart redraws instantly without any page reload.

Once you have a feel for the data, head down to the Train a Model panel. Choose an algorithm, set how much of your data to hold back for testing, tune the hyperparameters, and hit Train. The tool will sweep through a range of hyperparameter values automatically and show you a hyperparameter sweep chart so you can see exactly where performance peaks and where the model starts to overfit. Below that is the Decision Boundary Explorer, which draws the regions of the feature space where the model would predict each class, overlaid with your actual data points, so you can see not just how accurate the model is but exactly where and why it gets things wrong. Hover any point to see its true label, what the model predicted, and the feature values that led to that decision.

A good workflow is to open the correlation matrix first, find a feature pair that separates the classes, confirm it in the scatter plot, then train a model and open the boundary explorer on that same pair.
