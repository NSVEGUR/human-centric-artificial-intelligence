Optimization Objective

From the lecture:

arg min   1⁄n   ∑i=1n   ℓ(f(xi), yi) + λΩ(f)

How we use it in this tool:

min (1 − acctest) + λ · Ω(f)

(1 − acctest) Replaces the average 0-1 loss — it equals 0 when the model is perfect and 1 when it gets everything wrong. We evaluate this on the test set as specified in Task 2.
λ The regularization parameter you control with the slider. Higher λ means we care more about keeping the model simple.
Ω(f) For Decision Trees: number of leaves (more leaves = more complex). We normalize this to [0,1] so it stays on the same scale as the error.

Task 4: Counterfactual Explanations

What are Counterfactual Explanations?

Counterfactuals answer the question: "What would need to change for the model to predict a different class?" They help you understand what minimal changes to a penguin's measurements would result in it being classified as a different species.

Method: We sample many points near the original, find those predicted as the target class, and rank them by MAD-weighted L1 distance (smaller = more similar to original).

Task 5: Global Feature Effects (PDP & ALE)

Understanding Feature Effects

These plots show how each feature globally affects the model's predictions across all samples.

PDP (Partial Dependence Plot)

Shows the average effect of a feature on the prediction. For each feature value, we average the predictions over all samples.

PDP(xs) = 1/n ∑i f(xs, xc(i))

ALE (Accumulated Local Effects)

More robust when features are correlated. Computes local effects by looking at how predictions change within small intervals.
