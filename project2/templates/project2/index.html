Optimization Objective

From the lecture:

arg min   1⁄n   ∑i=1n   ℓ(f(xi), yi) + λΩ(f)

How we use it in this tool:

min (1 − acctest) + λ · Ω(f)

(1 − acctest) Replaces the average 0-1 loss — it equals 0 when the model is perfect and 1 when it gets everything wrong. We evaluate this on the test set as specified in Task 2.
λ The regularization parameter you control with the slider. Higher λ means we care more about keeping the model simple.
Ω(f) For Decision Trees: number of leaves (more leaves = more complex). We normalize this to [0,1] so it stays on the same scale as the error.

Accuracy vs Complexity Tradeoff

All trained models plotted by complexity (x) and accuracy (y). The red star shows the currently selected model. Move the λ slider to see the selection change.

Confusion Matrix

Shows which species the model confuses with which. A perfect model has all predictions on the diagonal. Darker = more predictions in that cell.

Train vs Test Accuracy Gap

A large gap between train and test accuracy indicates overfitting — the model memorized the training data but struggles on new data. Try increasing λ to reduce overfitting by choosing a simpler model.
